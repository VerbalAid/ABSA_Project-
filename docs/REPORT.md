# ABSA mT5 — Minimal Experiment (Synthetic Spanish)

This repo was intentionally reduced to a **small, single experiment**: generate synthetic Spanish ABSA data, train mT5 on it, and evaluate on real M‑ABSA Spanish test.

---

## Project files (minimal)

- `absa/` — training/eval/data code
- `scripts/experiment.py` — single CLI to run everything
- `config/experiment.yaml` — single config for the synthetic Spanish experiment
- `requirements.txt` — dependencies
- `docs/REPORT.md` — this document

---

## Task

Input: review sentence  
Output: triplets \((\text{aspect term},\ \text{aspect category},\ \text{sentiment})\)

M‑ABSA line format:

```
<sentence>####[[aspect_term, aspect_category, sentiment], ...]
```

Sentiment must be one of: `positive`, `negative`, `neutral`.

---

## Metrics

Eval reports:
- **micro_precision**
- **micro_recall**
- **micro_f1**

All computed over **exact triplet matches**.

## Synthetic data pipeline (end-to-end)

### 1) Extract allowed categories (ground truth)

Hotel (Spanish):

```bash
python scripts/experiment.py extract-categories --domain hotel --language es --split train --out data/categories_hotel_es.txt
```

Restaurant (Spanish):

```bash
python scripts/experiment.py extract-categories --domain restaurant --language es --split train --out data/categories_restaurant_es.txt
```

### 2) Generate raw synthetic lines (Ollama)

Prereq: Ollama running and model pulled:

```bash
ollama pull mistral
```

Generate raw hotel:

```bash
python scripts/experiment.py gen-ollama \
  --domain hotel \
  --categories data/categories_hotel_es.txt \
  --out data/raw_synth_es_hotel.txt \
  --n 800 \
  --model mistral
```

Generate raw restaurant:

```bash
python scripts/experiment.py gen-ollama \
  --domain restaurant \
  --categories data/categories_restaurant_es.txt \
  --out data/raw_synth_es_restaurant.txt \
  --n 800 \
  --model mistral
```

### 3) Validate and clean into training files

Clean hotel:

```bash
python scripts/experiment.py validate \
  --in data/raw_synth_es_hotel.txt \
  --out data/synthetic_es_hotel_train.txt \
  --allowed data/categories_hotel_es.txt
```

Clean restaurant:

```bash
python scripts/experiment.py validate \
  --in data/raw_synth_es_restaurant.txt \
  --out data/synthetic_es_restaurant_train.txt \
  --allowed data/categories_restaurant_es.txt
```

These outputs are the exact files referenced by `config/experiment.yaml`.

# ABSA mT5 — Simple Experiment Report (Single Source of Truth)

This file is the **only documentation you need** for this project: what the code does, how to run it, what experiments we ran, what failed and why, what was fixed, and how to interpret the results.

## 1) Goal (what we are trying to prove)

We want to evaluate **cross-lingual transfer** and the value of **synthetic Spanish training data** for ABSA triplet extraction.

### Core question (research)

- If we train on **synthetic Spanish** hotel+restaurant data (generated by an LLM) and evaluate on **real M‑ABSA Spanish test**, how good is the resulting model?
- How does it compare against:
  - **English baseline**
  - **English→Spanish cross-transfer**
  - **Spanish fine-tuned on real M‑ABSA Spanish**

## 2) Task & data format

### Task

Input: a review sentence.  
Output: a set of ABSA triplets:

\[
(\text{aspect term},\ \text{aspect category},\ \text{sentiment})
\]

### Dataset

M‑ABSA on Hugging Face: `Multilingual-NLP/M-ABSA`.

Each example is a single line:


## 5) Metrics (what “good” means)

Evaluation reports:

- **micro_precision**
- **micro_recall**
- **micro_f1**

All computed over **exact triplet matches**: a predicted triplet counts as correct only if
`(term, category, sentiment)` matches a gold triplet exactly.

Interpretation:
- **Precision > Recall**: model is conservative (predicts fewer triplets; misses many).
- **Recall > Precision**: model is aggressive (predicts more; more false positives).
- **F1** summarizes both.

## 6) Experiments (models we care about)

Even though this repo is now “one experiment”, it’s useful to understand the baselines:

### Model A — English baseline (EN → EN)

- Train on EN hotel, test on EN hotel.
- Previously observed test: **micro_f1 ≈ 0.3842**.

### Model B — Cross-lingual transfer (EN → ES test)

Evaluate the same EN-trained checkpoint on Spanish hotel test.

Observed:
- **micro_precision = 0.2458**
- **micro_recall = 0.1676**
- **micro_f1 = 0.1993**

Interpretation:
- Spanish is ~52% of English baseline (\(0.1993 / 0.3842 \approx 0.52\)).
- Precision > recall suggests the EN model is conservative on Spanish.

### Model C — Spanish fine-tune on real M‑ABSA (ES → ES)

We ran a **low-VRAM** Spanish training successfully on a ~8GB GPU using:
- `optim: adafactor` (optimizer-state memory saver)
- batch size 1 + gradient accumulation
- gradient checkpointing

Training finished and wrote checkpoint shards for `checkpoints/spanish_hotels`.

### Model D — Synthetic Spanish (main research)

Train on synthetic Spanish hotel+restaurant data, evaluate on real M‑ABSA Spanish test.

This is what `config/experiment.yaml` is for.

## 9) Practical issues we hit (and what fixed them)

### 9.1 Training OOM on ~8GB GPU (Spanish fine-tune)

- **Where it crashed**: Adam optimizer state init (`exp_avg` allocation) at step 0.
- **Fix**: use **Adafactor** + smaller batch + gradient checkpointing + shorter lengths.

### 9.2 Eval OOM on small GPUs

- **Fix**: force CPU eval with:

```bash
EVAL_USE_CPU=1 CUDA_VISIBLE_DEVICES=""
```

### 9.3 HF dataset cache lock errors

- **Fix** (operational): delete stale `.lock` files and ensure cache dir is writable.

### 9.4 Overflow errors during decode/metrics

- **Fix** (code): cast/clamp token IDs before decoding.
