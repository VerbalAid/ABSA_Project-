# ABSA Triplet Extraction: English & Spanish Hotel Reviews
**Darragh Kerins and Ander Peña** · UPV/EHU · 2025

## What is ABSA?

Aspect-Based Sentiment Analysis (ABSA) is a fine-grained form of sentiment analysis that goes beyond asking "is this review positive or negative?" Instead, it identifies *what* is being talked about and *how* the reviewer feels about each specific thing.

For a sentence like:

> *"The room was spotless but the check-in took forever."*

A standard sentiment model might call this mixed or neutral. ABSA extracts the structure underneath:

| Aspect term | Category | Sentiment |
|---|---|---|
| room | `ROOM#CLEANLINESS` | positive |
| check-in | `HOTEL#SERVICE` | negative |

Each of these three-part extractions is called a **triplet**: *(aspect term, aspect category, sentiment polarity)*. The aspect term is the exact span from the review text; the category is drawn from a fixed label schema; and the polarity is positive, negative, or neutral.

This project trains a multilingual seq2seq model to extract all triplets from a hotel review in a single pass, and investigates whether LLM-generated synthetic training data can replace human-annotated or machine-translated data for Spanish.

## Results at a Glance

| Model | Training Data | Test Set | Precision | Recall | F1 |
|-------|--------------|----------|-----------|--------|----|
| A – English baseline | M-ABSA EN (real) | EN | 60.71% | 46.21% | **52.48%** |
| B – Spanish translated | M-ABSA ES (machine-translated) | ES | 50.76% | 37.60% | 43.20% |
| C – Spanish synthetic | LLM-generated (Claude) | ES | 17.83% | 15.06% | 16.33% |
| D – Spanish combined | Translated + synthetic | ES | 51.07% | 43.26% | **46.84%** |
| Mistral-7B zero-shot | — | ES | 7.05% | 1.81% | 2.88% |

**Key finding**: Synthetic data alone (Model C) substantially underperforms machine-translated data (Model B), but combining both (Model D) produces the best Spanish result — outperforming the translation-only baseline by **3.64 F1 points**, driven by improved recall.

Zero-shot Mistral-7B (7B parameters) is decisively outperformed by fine-tuned mT5-small (300M), confirming that task-specific training on modest in-domain data beats prompting a larger model for structured extraction with a fixed label schema.

## Task

Each model extracts structured opinion **triplets** from hotel reviews:

Each triplet contains: **aspect term** (exact span from the review) · **aspect category** (from the M-ABSA label schema) · **sentiment polarity** (positive / negative / neutral).

Evaluation uses **exact-match micro-F1**: all three elements must match the gold annotation exactly.

## Data

| Split | Examples | Source |
|-------|----------|--------|
| English train / dev / test | 1,255 / 308 / 584 | M-ABSA `hotel_en` (real annotations) |
| Spanish translated train / dev / test | 1,255 / 308 / 584 | M-ABSA `hotel_es` (machine-translated) |
| Spanish synthetic train | ~1,080 | LLM-generated via Claude (see below) |

M-ABSA is available on HuggingFace at `Multilingual-NLP/M-ABSA`.

### Synthetic Data Generation

Synthetic Spanish training examples were generated by prompting Claude with the full real M-ABSA Spanish training split as few-shot context. Each batch of 200 generated lines was validated against three criteria: correct parse format, category labels drawn exclusively from the real training set, and at least one valid triplet per line. Four batches yielded ~1,080 clean examples after filtering.

---

## Models and Training

All fine-tuned models use **mT5-small** in a seq2seq framing: given a review sentence, the model generates all triplets as a single structured string.

**Training config**: 5 epochs · lr 3×10⁻⁴ · batch size 2 · bf16 where supported · `tie_word_embeddings=False`
